## Anti-Patterns Verdict

**LLM assessment:** This does not read as generic AI slop. The system has a specific child-game vocabulary: chunky outlines, toy-like controls, direct German copy, and simple boards. The main risk is not sameness; it is inconsistency. The old `formenmalen` surface is still visually prominent on the homepage and can accidentally define the brand.

**Deterministic scan:** `detect.mjs --json index.html projects/bunte-bausteine/index.html projects/mastermind/index.html projects/formenmalen/index.html` returned `[]`. No bundled detector findings.

**Visual overlays:** Browser screenshots and DOM measurements were collected. User-visible detector overlays were not injected because the available browser evaluation surface is read-only for mutation, so there was no reliable script-injection path.

## Overall Impression

The strongest version of the site is already visible in `bunte-bausteine` and `mastermind`: warm paper background, cyan game surfaces, dark tactile outlines, and clear action colors. A tasteful refinement pass should make that direction unmistakable, especially on the homepage and mobile.

## What's Working

- The core vocabulary is memorable: dark outlines, offset shadows, rounded game cells, and saturated action colors feel physical without becoming over-decorated.
- The copy is refreshingly direct. Labels like `Neues Spiel`, `Bewerten`, `Reihe leeren`, and `Alles löschen` are literal and child-readable.
- Mastermind already has the best interaction model: status text, disabled invalid actions, clear row structure, and recovery messages.

## Priority Issues

### [P1] Homepage direction conflict

**Why it matters:** The homepage still foregrounds the older `formenmalen` visual language in the hero stack and first project card. That contradicts the new product/design docs and makes the future direction less clear.

**Fix:** Put `bunte-bausteine` and `mastermind` visually first. Move `formenmalen` behind them in the hero stack and later in the project order, or mark it subtly as older until it is refreshed.

**Suggested command:** `$impeccable layout index.html`

### [P1] Mobile Bunte Bausteine controls and cells are too tight

**Why it matters:** At a 390px viewport, the tool buttons clip horizontally and the grid cells shrink to about 26px. That breaks the product principle of large, forgiving controls for a young user.

**Fix:** Let the action buttons wrap on mobile, group color buttons into two rows or a scroll-safe palette, and choose a mobile grid treatment that preserves tappable cells instead of forcing all 12 columns into one narrow row.

**Suggested command:** `$impeccable adapt projects/bunte-bausteine`

### [P2] Mastermind needs one more layer of state guidance

**Why it matters:** The game looks strong, but a first-timer has to infer the flow: start, choose colors, fill one row, understand black/white feedback. The interface can teach that without adding a tutorial.

**Fix:** Highlight the active row, make the status text say the next action, and add a tiny feedback legend for black/white pins after the first guess or near the board.

**Suggested command:** `$impeccable clarify projects/mastermind`

### [P2] The shared component system is not fully normalized

**Why it matters:** The docs now define the intended direction, but the homepage still uses slightly different paper/rose values and a more landing-page-like hero treatment. Formenmalen is visibly from another generation.

**Fix:** Align homepage colors/radii/shadow rhythm to `DESIGN.md`; defer the full Formenmalen refresh but stop letting it set the visual tone.

**Suggested command:** `$impeccable polish index.html`

## Persona Red Flags

**Jordan (First-Timer):** Mastermind has clear labels, but the feedback pins are not explained. Jordan may fill a row, receive black/white dots, and not know what they mean.

**Sam (Accessibility-Dependent User):** Color buttons have ARIA labels, which is good, but the visual experience still relies heavily on color. Bunte Bausteine cells all announce similar labels, and future feedback should avoid color-only meaning.

**Casey (Distracted Mobile User):** Bunte Bausteine is the concern. Tool buttons clip at 390px, and the 12-column grid creates small tap targets. Mastermind is much stronger on mobile.

**Elina (Young Player):** The strongest screens feel playable immediately. The old yellow turtle game feels more like a prototype and should not be the first visual promise of the collection.

## Minor Observations

- Add a small home/back link to each game page eventually; currently the game pages are dead ends.
- Disabled Mastermind buttons are understandable, but the muted text on pastel fills feels a little muddy.
- The homepage CTA `Direkt losbauen` correctly points to Bunte Bausteine; the project order should support that same priority.
- Bunte Bausteine `Alles löschen` is low-stakes, but an undo or immediate status response would make it feel more forgiving.

## Questions to Consider

- What should the homepage promise first: the collection, or the two best current games?
- Should `formenmalen` be presented as a legacy app until it is updated, or simply moved behind the stronger two?
- Does Bunte Bausteine need all 12 columns visible on mobile, or should mobile prioritize tappable cells over seeing the whole board at once?
